2.     Experimental Hutch Shutter

 

2.1     The experimental hutch shutter can be opened and closed by clicking on the Shutter1 OPEN or CLOSE button on the GDA terminal.

 

 

Check the indicator for the Experimental Shutter on the beamline synoptic has changed.

 

2.2     Or the following commands can be used:

Opening the shutter

 

pos ehshutter Reset

 

pos ehshutter Open

 

Check that the indicator for the Experimental Shutter has changed to Open on the beamline synoptic.

 

Closing the shutter

 

pos ehshutter Closed

 

3.     beam monitor

 

The beam monitor command will monitor the value of the ion chamber Ic4. If Ic4 falls below a threshold value there is no x-ray beam present, because the experimental shutter is not open or the synchrotron has lost beam, the scan will be paused.

 

The warning message 'SCAN PAUSED. WAITING FOR BEAM ON' will appear in the GDA window.

 

When beam is returned the scan will resume (step scan only)

 

For a constant velocity scan the scan cannot simply be paused and restarted in this manner. Instead the current cvscan will be stopped completely if the beam monitor detects no beam, and restarted from the beginning when beam is returned.

 

3.1     Beam monitor commands

 

bm.on() - to turn on the beam monitor facilty

 

bm.off() - to turn it off

 

To query the state of the monitor: bm.isMonitorOn()

Return: 1 = yes

 0 = no

            spin.off()

            (Sets the sample spinner speed to 3000 rpm)

 

4.3     Query the spin speed set:

 

            spin.getSpeed()

5.     Scans and position motors

Constant Velocity Scanning Commands

5.1     Constant velocity scan (cvscan)

 

cvscan [motor] [start] [time in seconds] [number of repeats]

 

If no repeat number is specified the default is one.

 

Example: cvscan tth 10 3600

(2q cvscan, starts at 10° with a total counting time of 1 hour)

 

Example: cvscan tth 10 1800 3

(Repeat three times 2q cvscans, starting at 10° with a total counting time of ½ hour each)

 

5.2     Constant velocity scanning of 2q and spos

 

The sample position can be continuously scanned during a 2q cvscan

 

cvscan tth [start] time spos [start] [finish]

 

Example: cvscan tth 10 3600 spos -20 0

A 1 hour 2q and spos combined cvscan with 2q stating at 10° and the sample position moving between -20 and 0.

 

Note that -20 and 0mm are the upper and lower limits of spos.

 

5.3     Constant velocity scans with sample stage translation motor (stagescan)

 

Separate 2q cvscans are carried out for each value of spos (different to 5.2)

 

stagescan spos [start] [finish] [step] tth [start] [time in seconds]

 

Example: stagescan spos -2 2 1 tth 10 1800

(Stepping sample spinner stage by 1mm from -2 to +2mm with a 2q cvscan, starting at 10° with a total counting time of 1/2 hour each, at each step of spos)

 

5.4     Constant velocity scan with theta rocking

 

When the phenix cryostat is in use the theta circle of the diffractometer on which it is mounted can be oscillated during a 2q cvscan.

 

cvscan tth [start] [time] theta [start] [stop]

 

Example: cvscan tth 5 600 theta -5 5

A 10 minute 2q cvscan scan will start from 5 degrees, during the scan theta will be oscillated between -5 and 5 degrees

 

The minimum time for a cvscan with theta rocking is 10 minutes (600s)

Step Scanning Commands

5.5     Single motor or step scan (scan)

 

scan [motor] [start] [finish] [step] [detector] [count time]

 

Example: scan tth 10 20 0.005 Io 2

(2q scan will start at 10° and finish at 20° with 0.005° step and 2 second per step measuring the intensity in Io monitor)

 

 

5.6     Multi-motor concurrent scans

 

            scan [motor1] [start1] [finish1] [step1] [motor2] [start2] [step2] [detector] [count time]

 

            Example: scan tth 10 30 0.002 theta 5 0.001 mac15 1

(q-2q  concurrent scan, will start at 10° and finish at 30° with 0.001° step and 1 second per step measuring the intensity in MAC15 detector)

 

5.7     Multi-motor nested or grid scans

 

            scan [motor1] [start1] [finish1] [step1] [motor2] [start2] [stop2] [step2] [detector name] [time]

 

            Example: scan tth 10 30 0.002 theta 5 10 0.001 mac15 1

(q-2q nested scan, will start at 10° and finish at 30° with 0.001° step for tth, and at each tth position theta will scan from 5° to 10° with 0.001° step, and at each theta position mac15 detector will measure scattering intensity for 1 second.)

 

6.     Robotic Arm

 

6.1     Manual Sample Change Commands

 

sample.start( )  to start/switch on robot

 

sample.recover( )  to recover sample and place it in the recovery tray.

This command must be used when the robot was stopped in an emergency or if user

dropped the interlock (Exp. Hutch) while the robot is in action.

 

pos sample [sample number]

 

sample.clearSample( )  to clear sample from the spinner and replace it to the original slot on the cassette

 

sample.finish( )  completion, to switch off the robot

 

sample.stop( )  stop robot in an emergency

 

6.2     Multiple sample scans (robotscan)

 

robotscan sample [1st sample] [last sample] [step] tth [start] [time]

 

Example:  robotscan sample 1 21 1 tth 10 1800

Note: the scanning of tth is in cvscan mode

 

 

6.3     Multiple sample scans with sample position translation at each sample

 

robotscan sample [1stsample] [lastsample] [samplestep] spos [startpos] [stoppos] [posstep] tth [start] [time]

 

6.4     Sample Info File

 

A file 'SampleInfo.xls' has been created in the processing folder within your data directory. This is an excel spreadsheet in which you can enter information about the samples loaded onto the robot carousel. To edit the file open the OpenOfficeCalc program on one of the beamline linux workstations (ws001 or ws002) then open file /dls/i11/data/YEAR/PROPOSALNUMBER/processing/SampleInfo.xls. The file can then be edited with your sample information. When completed save as excel format .xls file.

 

The file must then be loaded into gda. From the GDA terminal:

loadfile /dls/i11/data/YEAR/PROPOSALNUMBER/processing/SampleInfo.xls

 

During a robot scan the information for the current sample number will be written to the data header file.

 

Remember to update and reload the file if you change the position of your samples on the carousel!

 

6.5     Robot error codes

 

2010        - diffractometer move not completed

2450    - interlock problem

3540    - Robot interlock key missing

2070    - servo not 'on'

 

We have implemented error code to error message mapping. If you see an error code without message attached, please report to us.

 

7.     oxford Cryostream operation

 

Note: The cryostream temperatures are in KELVIN (K)

 

7.1     Manual Temperature Control Commands

 

ocs.start()  run the startup process of the cryostream

 

pos ocs   get Current Temperature in Kelvin

 

ocs  get current temperature in Kelvin

 

pos ocs 270  set temperature to 270K

 

ocs.getRampRate()  get the current temperature ramp rate in K/hour

 

ocs.setRampRate(180)  set the temperature ramp rate to 180K/hour. The maximum ramp rate is 360 K/hour

 

ocs.stop()  stop and hold the temperature immediately

 

ocs.end()  run the shutdown process of the cryostream.

 

7.2     Temperature Scanning

 

tempscan ocs [startT] [finishT] [Tstep] [rate] [wait time in s] tth [startangle] [cvscan time]

 

The cryostream will scan the temperature from start T to finish T carrying out a 2q cvscan at intervals of T step, the temperature will ramp between values at the specified rate. GDA will wait for the given time once the temperature is reached before starting the cvscan. The wait time must be an integer number of seconds.

 

Example:

tempscan ocs 200 210 5 360 300 tth 10 3600

1 hour cvscans will be performed at 200, 205 and 210K. The cryostream will ramp between these temperatures at 360K/hour and GDA will wait for 5 minutes (300s) once the temperature is reached before starting the 2q  cvscan.

 

The tempscan command can be used for heating or cooling runs, but in both cases the values of T step should be inputted as positive i.e.

 

Heating ramp:

tempscan ocs 200 210 5 360 300 tth 10 3600

Cooling ramp:

tempscan ocs 210 200 5 360 300 tth 10 3600

 

8.     Cyberstar Hot-Air blower operation

 

Note: The hot-air blower temperatures are in Celsius (°C)

8.1     Manual Temperature Control Commands

pos csb   get Current Temperature in Degress Celsius

 

pos csb 500  set temperature to 500°C

 

The ramp rate is 5°C/min (this cannot be set)

 

To turn off the hot-air blower set the temperature to room temperature and allow to ramp down before switching off. i.e. pos csb 23

 

8.2     Temperature Scanning

 

tempscan csb [startT] [finishT] [Tstep] [wait time in s] tth [startangle] [cvscan time]

 

As for the cryostream (§7.2), but the rate cannot be set.

9.1      Manual Temperature Control Commands

pos pcs   get Current Temperature in Kelvin

 

pos pcs 50  set temperature to 50K

10.     Rebinning Data

 

9.1       After a cvscan scan

 

            After a cvscan the data will automatically be processed and rebinned to a step size of 0.001 deg. The processed data file will be saved in the processing directory of your data directory and given the file name datafilenumber_red.dat i.e. a raw data file 3051.dat will be processed and saved as follows:

                       

/dls/i11/data/YEAR/PROPOSALNUMBER/processing/3051_red.dat

 

9.2       Rebinning data offline

 

For rebinning a raw data file from a cvscan to a different step size after the scan is completed. This should be done from the scripts directory (/dls/i11/scripts) in a linux terminal window

 

To rebin a data file:

 

                        ./rebin.py /dls/i11/data/YEAR/PROPOSALNUMBER/[datafile] [stepsize] [comment]

 

                        Example: ./rebin.py /dls/i11/data/2008/ee0/3051.dat 0.002 1

            (sum the raw data from file 3051 and rebin the data to a step size of 0.002 deg. The rebinned data file will then be called 3051_red_1.dat)

The rebinned data file will be saved in the processing directory within your data directory i.e.

 

/dls/i11/data/YEAR/PROPOSALNUMBER/processing/3051_red_1.dat

 

To rebin and sum multiple data files

 

            ./rebin.py -a "comma separated file list" /dls/i11/data/YEAR/PROPOSALNUMBER/[datafile] [stepsize] [comment]

 

            Example:

            ./rebin.py -a "4727, 4728" /dls/i11/data/2008/ee0/4726.dat 0.001

Rebins sums and normalises 4726, 4727 and 4728.

The rebinned data file is saved in your processing directory and called,  4726_4727_4728_red.dat

 

11.     Writing scripts

 

Any of the GDA commands can be written in a script form using the 'JythonEditor' window in the GDA panel. Click 'New' to start a new script. Enter the commands one on each line as you would for the terminal window.

 

Example:

pos ocs 100

sleep(300)

cvscan tth 5 1800

pos ocs 125

sleep(300)

cvscan tth 5 1800

tempscan 150 200 5 300 tth 5 900

 

The sleep(time in sec) command is used to tell GDA to wait for this amount of time after the previous command is completed before running the next line.

 

The script must be saved before running. To run a script press the 'run' button on the JytonEditor panel. A script can only be run if the button is highlighted. If it isn't then either the script is not saved or GDA is still busy running other commands.
